## 📁 仓库结构

当前仓库（`personal-hub`）的主要结构如下：

- `index.html`：个人首页 / 导航页，汇总入口到各个小游戏与项目
- `css/`
  - `style.css`：首页的全局样式（背景、导航、卡片等）
- `games/`
  - `archero_like/`：Roguelike 吸血鬼幸存者「代码飞升：全栈修仙传」
    - `index.html`：游戏页容器，只负责结构与资源引用
    - `css/style.css`：该游戏的专用样式（HUD、弹窗、卡片等）
    - `js/game.js`：核心游戏逻辑（GameLoop、实体系统、技能与数值等）
    - `assets/`：预留的静态资源目录（图片、音频等）
  - `2/`：预留的小游戏 2（占位，后续填充）
  - `3/`：预留的小游戏 3（占位，后续填充）

> 说明：随着后续增加更多小项目 / 小游戏，会继续在 `games/` 目录下扩展新的子目录，并在根目录 `index.html` 中增加入口卡片。

## 🎮 已有项目简介

### 1. 代码飞升：全栈修仙传（`games/archero_like`）

- 类型：**HTML5 Canvas Roguelike 生存射击**
- 玩法设定：
  - 选择不同「技术栈门派」（前端 / 后端 / 移动端），拥有差异化属性与敌人池
  - 自动射击 + WASD/方向键移动，持续击杀 Bug 获取 KPI 与技能
  - 通过卡牌式技能升级实现无限成长与数值叠加
- 技术亮点：
  - 轻量级 **Game Loop** 与状态机（Menu / Playing / Paused / Upgrade / GameOver）
  - 使用 **Emoji 作为贴图**，无需额外资源即可获得丰富视觉效果
  - 独立拆分的结构（HTML 负责结构、CSS 负责 UI、JS 负责游戏逻辑），便于后续维护与扩展
  - 预留 `assets/` 目录，后续可平滑替换为图片贴图版本

### 2. 小游戏 2（`games/2`）与 小游戏 3（`games/3`）

- 目前为占位目录，用于后续添加新的小游戏 / 交互实验。
- 首页 `index.html` 中已经预留入口卡片，未来可以直接接入对应项目。

## 🧭 使用方式

本仓库 = 纯静态站点，无需后端与复杂构建工具：

1. 克隆或下载本仓库到本地。
2. 直接用浏览器打开根目录的 `index.html` 即可访问首页。
3. 通过首页中的「小游戏合集」卡片进入各个项目，例如：
   - 程序员肉鸽：`./games/archero_like/index.html`

也可以将整个目录托管到任意静态托管服务（GitHub Pages / Vercel / Netlify 等），作为个人作品集主页。
