## 📁 仓库结构

当前仓库（`personal-hub`）的主要结构如下：

- `index.html`：个人首页 / 导航页，汇总入口到各个小游戏与项目
- `css/`
  - `style.css`：首页的全局样式（背景、导航、卡片等）
- `games/`
  - `archero_like/`：国风修仙 Roguelike「问道：凡人修仙」
    - `index.html`：游戏入口
    - `css/style.css`：修仙风格专用样式（水墨、流金 UI）
    - `js/game.js`：核心逻辑（境界成长、宗门体系、实体组件）
    - `assets/`：预留的静态资源目录
  - `magic_tower/`：经典策略闯关游戏「魔塔：经典50层版本」
    - `index.html`：游戏入口
    - `css/style.css`：复古魔塔风格样式
    - `js/script.js`：核心逻辑（地图系统、战斗算法、物品交互）

> 说明：随着后续增加更多小项目 / 小游戏，会继续在 `games/` 目录下扩展新的子目录，并在根目录 `index.html` 中增加入口卡片。

## 🎮 已有项目简介

### 1. 问道：凡人修仙（`games/archero_like`）

- 类型：**HTML5 Canvas 国风修仙 Roguelike**
- 故事背景：
  - 讲述一名现代人穿越到修仙界，以凡人之躯逆天改命的故事。
  - **多宗门设定**：计划包含蜀山、青城、魔宗等八大门派（目前首发开放【蜀山剑派】）。
- 玩法特色：
  - **境界体系**：从练气到真仙的指数级成长，还原小说中的修真阶梯。
  - **肉鸽构筑**：每次突破境界触发"神念择道"，三选一构筑专属神通组合。
  - **动态战斗**：满屏的飞剑与弹幕，结合 Canvas 粒子特效，带来流畅的战斗体验。
- 技术亮点：
  - **原生 Canvas 渲染**：无素材依赖，纯代码绘制动态飞剑与特效。
  - **ECS 架构雏形**：分离实体数据与逻辑，支持未来快速扩展新门派与技能。
  - **完全响应式**：暂时只支持PC网页。

### 2. 魔塔：古老的誓约（`games/magic_tower`）

- 类型：**HTML5 经典策略闯关游戏**
- 故事背景：
  - 艾尔迪亚大陆纪元 1204 年，黑暗领主苏醒，将公主囚禁于无尽之塔。
  - 你是王国最后的希望——一位年轻的皇家骑士。
- 玩法特色：
  - **10层地牢探索**：从底层一路爬升，通过钥匙、楼梯机制探索迷宫。
  - **数值策略**：战斗伤害计算透明，需要合理分配血量、攻击、防御数值。
  - **怪物图鉴系统**：点击怪物即可查看属性与战斗预测，避免盲目送死。
  - **资源管理**：三色钥匙、血瓶、宝石等道具构成关键的策略抉择。
- 技术亮点：
  - **纯原生实现**：零依赖，无图片素材，通过 CSS + Emoji 渲染所有元素。
  - **回合制战斗算法**：完整实现了经典魔塔的伤害计算与回合推演。
  - **开场滚动字幕**：复古风格的沉浸式剧情介绍。

### 3. 党国将领风云录（`games/poker_card`）

- 类型：**历史题材卡牌互动图鉴 / 对战游戏**
- 故事背景：
  - 基于一份流传的“国民党将领扑克天梯榜 (1949版)”，重现动荡年代的军政格局。
  - 包含 54 位关键历史人物，从最高领袖到基层悍将，等级森严，派系林立。
- 玩法特色：
  - **天梯阅兵**：按 `Joker > 2 > A...` 的层级展示将领序列，支持整行流水灯翻牌交互。
  - **随机对决 (War)**：内置比大小对战模式，体验不同将领之间的“官大一级压死人”或“以下克上”。
  - **派系视图**：可按【王牌核心】、【军政核心】、【财经后继】等派系重组展示。
- 技术亮点：
  - **民国档案 UI**：纯 CSS 复刻青天白日卡背与深色金属质感卡面，无需图片素材。
  - **3D 交互**：丝滑的 CSS3 3D Transform 翻转动画与光效。
  - **数据驱动**：结构化的 JSON 数据源，支持轻松扩展或替换人物头像。

## 🧭 使用方式

本仓库 = 纯静态站点，无需后端与复杂构建工具：

1. 克隆或下载本仓库到本地。
2. 直接用浏览器打开根目录的 `index.html` 即可访问首页。
3. 通过首页中的「小游戏合集」卡片进入各个项目，例如：
   - 问道：`./games/archero_like/index.html`
   - 魔塔：`./games/magic_tower/index.html`

也可以将整个目录托管到任意静态托管服务（GitHub Pages / Vercel / Netlify 等），作为个人作品集主页。
